# 마이링크(My Link) - 기능 정의서(PRD)

## 1. 프로젝트 개요
- **프로젝트 명**: 마이링크 (My Link)
- **목적**: 
  - 크리에이터 등 다양한 플랫폼에서 활동하는 사용자가 자신의 포트폴리오, 작업물, SNS 링크를 하나의 단일 링크로 통합하여 공유할 수 있도록 지원.
- **대상 사용자**: 
  - 유튜브, 인스타그램, 틱톡 등 여러 채널을 운영하며 링크를 모아서 보여주길 원하는 크리에이터 및 일반 사용자.
- **주요 기술 및 디자인 스택**:
  - **인증 및 백엔드**: Firebase (파이어베이스 Auth, Firestore)
  - **UI/디자인 시스템**: shadcn/ui

---

## 2. 필수 기능 목록 (Must-have)

1. **사용자 계정 관리**: **Firebase 기반 구글(Google) 소셜 로그인** (단일 인증 방식).
2. **프로필 관리 (인라인 편집 기반)**: 
   - 닉네임(`displayName`), 실제 사용자 이름(`username`), 짧은 소개글 설정 및 수정.
   - **URL 슬러그(Slug)**: 사용자의 `displayName` 값이 고유 URL의 슬러그로 사용됨 (`mylink.com/닉네임`).
   - 페이지 상단에는 방문자들에게 **유저 네임(username)**이 실제 이름이나 명칭으로 노출됨.
   - *(※ 프로필 이미지 업로드 기능 제외)*
3. **링크 관리 (인라인 편집 CRUD)**: 
   - 텍스트 링크 추가, 수정, 삭제 기능. (수정 시 화면 이동 없이 **인라인 편집(Inline Edit)** 적용)
   - **파비콘(Favicon) 구글 API 연동**: 링크 등록 시 Google Favicon API를 호출하여 해당 사이트의 파비콘을 자동으로 표시.
4. **기본 화면 제공 및 뷰어**: 
   - 별도의 테마 기능(색상 변경 등) 없이 shadcn/ui 기반의 단일화된 깔끔한 기본 UI 제공.
   - 방문자가 링크들을 확인할 수 있는 퍼블릭 프로필 페이지 뷰.
5. **링크 클릭 조회수 (추후 확장)**: 
   - 나중에 개별 링크를 방문자가 몇 번 클릭했는지 파악할 수 있는 조회수 통계 기능 추가 예정.

*(※ 제외된 기능: 링크 활성화 토글, 순서 변경, 관리자 모바일 프리뷰, 이미지 업로드 직접 지원, 테마 커스터마이징, 별도의 URL 핸들 아이디 설정)*

## 3. 데이터베이스 모델링 구조 (Firebase Firestore)

안정성과 확장성을 고려하여 Firestore의 **서브 컬렉션(Sub-collections)** 기반으로 구조를 설계합니다.

- **유저 정보 필수 필드**: 데이터베이스 저장 시 접속 주소가 될 닉네임(`displayName`)과 화면에 표시될 유저 네임(`username`)이 데이터에 반드시 포함되어야 합니다.
- **기본 논리적 구조 (예시)**:
  - `users` (Collection) 
    - `[user_uid]` (Document): `displayName` (사용자 닉네임이자 URL 경로), `username` (실제 화면에 표시될 명칭/이름), 이메일, 소개글 등.
    - `links` (Sub-collection of `[user_uid]`): 사용자가 생성한 개별 링크 데이터들 저장.
      - `[link_id]` (Document): 목적지 URL, 링크 제목(Title), (추후 추가될) 클릭 조회수 카운트 등.

## 4. 각 기능의 상세 설명

### 4.1 사용자 계정 관리
- **구글 로그인 전용 시스템**: 별도의 이메일 가입 절차 없이 편의성을 위해 Google 소셜 인증만 단독 연동.
- **초기 닉네임(displayName) 자동 생성**: 최초 가입(로그인) 시, 사용자의 구글 지메일(Gmail) 주소에서 `@` 앞부분의 아이디를 추출하여 URL 슬러그로 사용될 `displayName`의 기본값으로 자동 설정. (예: `abcd12@gmail.com` -> `abcd12`)

### 4.2 프로필 관리 (인라인 편집 UI)
- **URL 슬러그 매핑**: 별도의 영문 ID(핸들)를 입력받는 과정 없이, 유저의 `displayName`(닉네임) 그 자체가 프로필의 접속 주소(Slug)로 활용됨 (`/displayName`).
- **유저 네임 분리**: 접속용 주소인 닉네임과 별개로, 방문자에게 가장 크게 보여질 실제 사용자 이름(`username`)을 분리하여 제공.
- **프로필 정보 수정**: 
  - 화면 상에서 유저 네임, 닉네임(URL 슬러그), 한 줄 소개글 텍스트 영역을 클릭하여 바로 수정할 수 있는 **인라인 편집(Inline Edit)** UI 채택. (모달이나 별도 페이지 이동 없음)

### 4.3 링크 설정 및 관리 (인라인 편집 UI)
- **링크 추가/수정/삭제**: 
  - 외부 주소(URL)와 링크 이름만 입력하여 리스트 형태로 관리.
  - 기존 등록된 링크를 수정할 때도 타이틀이나 URL 텍스트 영역을 클릭해 그 자리에서 즉시 수정(Inline Edit). 
- **Google Favicon API 파싱**: 데이터베이스에 직접 파비콘 이미지를 수집·저장하는 대신, 뷰어 렌더링 시 `https://www.google.com/s2/favicons?domain={입력된URL도메인}` 형식의 구글 API를 통해 빠르고 가볍게 아이콘을 노출.

## 5. 유저 인터페이스(UI) 주요 고려사항
- **반응형(Responsive) 뷰어**: 방문자 뷰어는 철저하게 모바일 기준(Mobile-first)으로 렌더링되며, 테마 선택 기능 없이 shadcn/ui 기반의 고정적이고 세련된 단일 디자인 유지.
- **관리자(어드민) 대시보드 편의성**: 데스크톱 및 모바일 관리 환경 모두에서 인라인 편집을 통해 가장 직관적으로 내 프로필과 링크 텍스트를 고칠 수 있도록 UX/UI 컴포넌트(입력 폼 전환 등) 최적화 배치.
