# 마이링크(My Link) - 사용자 시나리오 (User Scenarios)

모든 시나리오는 직관성을 높이기 위해 **"사용자는 [목적]하기 위해 [행동]한다"** 형식으로 기재되었습니다.

---

## 1. 방문자 (Visitor) 시나리오

### 방문 및 탐색
1. **프로필 접속**: 방문자는 소유자의 모든 SNS 및 포트폴리오를 한 곳에서 보기 위해 제공받은 `mylink.com/닉네임` URL로 접속한다.
2. **페이지 탐색**: 방문자는 찾고자 하는 목적지(콘텐츠)를 명확히 인지하기 위해 화면 상단의 [유저 네임(실제 이름)]과 [한 줄 소개글]을 읽고, 나열된 링크 블록들의 제목과 파비콘 아이콘을 둘러본다.
3. **오류 페이지(404)**: 방문자는 올바른 주소를 다시 확인하기 위해, 삭제되었거나 존재하지 않는 닉네임으로 접속했을 때 나타나는 "찾을 수 없는 페이지입니다" 안내 화면을 확인한다.

### 상호작용 (링크 이동 및 통계)
4. **링크 클릭 및 이동**: 방문자는 원하는 세부 목적지 웹사이트로 이동하기 위해 해당 링크 블록을 클릭(탭)하여 새 창에서 페이지를 연다.
5. **클릭 조회수 카운트**: 방문자가 링크를 클릭하면 시스템(마이링크)은 해당 링크의 인기도를 측정하기 위해 백그라운드에서 **해당 링크의 클릭 횟수를 1 증가**시킨다. (링크 클릭에 대해서만 카운트함).

---

## 2. 소유자 (Owner) 시나리오

### 인증 및 프로필 관리 (가입/로그인)
1. **구글 로그인**: 소유자는 마이링크 서비스를 간편하게 시작(또는 재접속)하기 위해 첫 화면에서 'Google로 로그인' 버튼을 클릭한다.
2. **초기 주소(슬러그) 확인**: 소유자는 자신의 기본 주소를 확인하기 위해, 로그인 성공 직후 구글 지메일 아이디의 앞부분을 기반으로 자동 생성된 닉네임(`mylink.com/닉네임`)을 확인한다.
3. **유저 네임 수정**: 소유자는 방문자에게 보여질 자신의 실제 이름이나 직함(예: '프론트엔드 개발자 아무개')을 변경하기 위해, 화면 상단의 유저 네임 텍스트를 마우스로 클릭하여 인라인(Inline)으로 텍스트를 수정한다.
4. **닉네임(URL) 수정**: 소유자는 자신의 마이링크 접속 주소를 변경하기 위해, 고유 닉네임 텍스트 영역을 직접 클릭하여 인라인으로 수정한다.
5. **소개글 수정**: 소유자는 자신을 한 줄로 어필하기 위해, 소개글 텍스트 영역을 클릭하여 인라인으로 내용을 기입 및 수정한다.

### 링크 생성, 수정, 삭제 (CRUD)
6. **링크 블록 추가**: 소유자는 새로운 주소(예: 새로 오픈한 포트폴리오 링크)를 등록하기 위해 '링크 추가하기' 버튼을 클릭하여 리스트 최상단(또는 빈 공간)에 새로운 링크 입력 블록을 생성한다.
7. **링크 정보 입력 (파비콘 연동)**: 소유자는 실제 페이지와 연결하기 위해 생성된 빈 블록 영역을 클릭하여 **목적지 URL**과 **링크 제목**을 인라인으로 입력한다. (이때 URL이 입력되면 Google Favicon API가 호출되어 자동으로 화면에 해당 사이트 아이콘이 뜬다.)
8. **링크 수정**: 소유자는 링크 텍스트에 발생한 오타나 변경된 URL 주소를 고치기 위해, 대상 링크의 텍스트 영역을 한 번 클릭해 그 자리에서 즉시(인라인) 내용을 수정한다.
9. **링크 삭제 (경고창)**: 소유자는 불필요하거나 만료된 링크를 제거하기 위해 대상 링크 블록의 '삭제(휴지통)' 버튼을 클릭하고, 실수로 지워지는 것을 방지하기 위해 생성된 팝업 경고창(Modal)에서 '확인'을 누른다.

### 공유 및 관리
10. **내 페이지 공유**: 소유자는 지인 및 팬들에게 완성된 내 페이지를 홍보하기 위해, 관리자 화면에 있는 '내 링크 복사(Share)' 아이콘을 클릭하여 URL을 클라이언트의 클립보드에 복사한다.
11. **통계 확인 (추후 기능)**: 소유자는 방문자들이 어떤 링크에 가장 관심이 있는지 분석하기 위해, 각 링크 항목 옆에 표시된 '클릭 수(조회수)' 데이터를 직접 확인한다.
